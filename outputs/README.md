# Outputs Directory

All output files are gitignored. They are generated by pipeline runs.

## Structure

```
outputs/
├── predictions/                          # Per-experiment retrieval + generation results
│   ├── dev_retrieval.json                # Retrieved chunks, contexts, scores
│   └── dev_predictions.json              # Generated answers + metadata
├── metrics/                              # Per-experiment evaluation results
│   ├── dev_kg2rag_fair_metrics.json      # Accuracy, retrieval, fairness, statistics
│   └── dev_kg2rag_fair_per_question.json # Per-question breakdown
├── logs/                                 # Timestamped pipeline logs
│   └── kg2rag_fair_dev_20260421_143000.log
└── pipeline_manifest.json                # Run tracking (stage status, artifacts, hashes)
```

## Pipeline Manifest

The `pipeline_manifest.json` tracks:
- Per-stage status (completed/skipped/failed), duration, artifact SHA-256 hashes
- Last run metadata (experiment name, split, version, total duration, timestamp)

This enables smart caching: stages 1-4 (PREPROCESS, KG_BUILD, DEMOGRAPHICS, INDEX) are skipped when their artifacts are already present.
